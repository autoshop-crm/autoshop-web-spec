# Этапы разработки и сроки (с активным использованием нейросетей)

Контекст: `DEVELOPMENT_ROADMAP.md`, `MICROSERVICES_ARCHITECTURE.md`, `CLIENT_MVP_NOW.md`, `PROJECT_OVERVIEW.md`, `MENTOR_BRIEF.md`.

## Ключевые этапы

1. Подготовка архитектуры и границ модулей (Core + 3 микросервиса)
   - Срок: 2-4 дня

2. Базовая инфраструктура (PostgreSQL, миграции, Docker Compose, базовый CI)
   - Срок: 3-5 дней

3. Core Monolith MVP (Client -> Vehicle -> Order, базовый Parts)
   - Срок: 3-5 недель

4. Интеграции и кросс-функции (UMAPI, Redis cache, Kafka events, RBAC/JWT)
   - Срок: 2-3 недели

5. Вынос вспомогательных сервисов (Notification, Auth, FileStorage) в рабочем минимуме
   - Срок: 3-5 недель

6. Клиентские приложения (Web кабинет + Android MVP)
   - Срок: 4-7 недель

7. Тестирование, стабилизация, демо-подготовка (Unit/Integration/E2E, фиксы)
   - Срок: 2-3 недели

## Быстрые оценки по объему

- Вертикальный MVP только `Client` (по `CLIENT_MVP_NOW.md`): 3-7 дней
- Backend MVP для демонстрации (Core + базовые интеграции): 6-10 недель
- Дипломный объем (Core + 3 микросервиса + Web + Android): 16-27 недель

## Главные тезисы

- Нейросети сильно ускоряют CRUD, DTO/Mapper, тестовые шаблоны и документацию.
- Узкие места остаются «человеческими»: архитектурные решения, интеграционные баги, безопасность, финальная стабилизация.
- Самый безопасный путь: сначала закрыть Core Monolith end-to-end, потом выносить сервисы.
