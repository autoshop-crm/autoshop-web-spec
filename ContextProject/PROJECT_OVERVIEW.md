## Введение

**CRM для Автосервиса** — это современная цифровая платформа, разработанная для полного управления бизнес-процессами автосервисных центров. Проект объединяет в себе передовые технологии, интуитивный пользовательский интерфейс и глубокую интеграцию с внешними системами для создания комплексного решения как для владельцев бизнеса, так и для конечных клиентов.

### Миссия проекта

Трансформировать традиционный автосервис в высокотехнологичный бизнес с прозрачными процессами, где каждый клиент получает персонализированный опыт, а владельцы бизнеса — полный контроль над операциями и аналитикой.

## Бизнес-концепция

### Проблемы, которые мы решаем

#### Для владельцев автосервисов:

1. **Хаотичный учет клиентов** — потеря контактов, истории обслуживания, невозможность построить долгосрочные отношения
2. **Неэффективное управление заказами** — бумажные записи, путаница в статусах, отсутствие приоритизации
3. **Проблемы с запчастями** — долгий поиск, неоптимальные цены, отсутствие контроля остатков
4. **Отсутствие аналитики** — невозможность понять прибыльность, эффективность механиков, сезонные тренды
5. **Низкая лояльность клиентов** — клиенты приезжают один раз и уходят к конкурентам

#### Для клиентов:

1. **Недоверие к автосервисам** — непрозрачное ценообразование, навязывание ненужных услуг
2. **Отсутствие контроля** — не знаешь, что делается с машиной, сколько это займет времени
3. **Забывчивость** — пропускают сроки ТО, замены масла, технических осмотров
4. **Потеря документов** — акты, чеки, гарантийные талоны теряются
5. **Сложность записи** — нужно звонить, ждать ответа оператора

### Наше решение

Мы создаем **экосистему**, где все процессы автоматизированы, прозрачны и удобны для всех сторон:

- **Web-панель** для администраторов, операторов и механиков
- **Мобильное приложение** для клиентов с "личным кабинетом" их автопарка
- **Облачное хранилище документов** (чеки, акты, фото ремонта)
- **Интеллектуальная система рекомендаций** на основе истории обслуживания
- **Программа лояльности** с геймификацией

## Целевая аудитория

### Основные сегменты

#### 1. Владельцы и управляющие автосервисов

**Профиль:**
- Малый и средний бизнес (1-5 постов)
- Техцентры среднего размера (5-15 постов)
- Сетевые автосервисы (15+ постов)

**Боли:**
- Нужен контроль за персоналом
- Требуется прозрачная финансовая аналитика
- Хотят автоматизировать рутинные операции
- Стремятся повысить лояльность клиентов

**Что получают:**
- Dashboard с ключевыми метриками (выручка, средний чек, загрузка)
- Отчеты по механикам, клиентам, популярным услугам
- Автоматические напоминания клиентам
- Систему лояльности "из коробки"

#### 2. Операторы и диспетчеры

**Профиль:**
- Принимают заказы
- Общаются с клиентами
- Формируют сметы
- Распределяют работу между механиками

**Боли:**
- Множество звонков одновременно
- Нужно вручную искать запчасти
- Сложно формировать сметы
- Забывают перезвонить клиентам

**Что получают:**
- Быстрый поиск клиентов и их истории
- Интеграцию с каталогами запчастей (UMAPI)
- Автоматический расчет смет с учетом скидок
- Систему напоминаний и уведомлений

#### 3. Механики и мастера

**Профиль:**
- Выполняют ремонт
- Нужен доступ к заказам и статусам
- Отчитываются о проделанной работе

**Боли:**
- Бумажные заказы-наряды теряются
- Нет быстрого доступа к истории автомобиля
- Сложно зафиксировать найденные проблемы

**Что получают:**
- Мобильный доступ к своим заказам
- Возможность добавлять фото/видео дефектов
- Обновление статусов в реальном времени
- Цифровые чек-листы для диагностики

#### 4. Конечные клиенты (владельцы автомобилей)

**Профиль:**
- Владельцы 1-3 автомобилей
- Возраст: 25-55 лет
- Ценят свое время и деньги
- Привыкли к цифровым сервисам

**Боли:**
- Не доверяют автосервисам
- Забывают о плановом обслуживании
- Теряют документы
- Не знают, куда обратиться с конкретной проблемой

**Что получают:**
- "Мой гараж" — вся информация о машинах в одном месте
- Напоминания о ТО, замене масла, страховке, техосмотре
- Прозрачное ценообразование и онлайн-сметы
- Программу лояльности с реальными скидками
- Рекомендации по обслуживанию на основе AI

## Ключевые возможности системы

### 1. Управление клиентами (CRM Core)

#### Карточка клиента
```
📋 Основная информация
├── ФИО, дата рождения
├── Контакты (телефон, email, Telegram)
├── Адрес
└── Дата регистрации

🚗 Автопарк клиента
├── Все зарегистрированные автомобили
├── VIN, госномер, марка/модель
├── Год выпуска, пробег
└── Страховка, ПТС

📊 История взаимодействий
├── Все заказы (дата, сумма, услуги)
├── Предпочтения (любимый механик, время визита)
├── Жалобы/отзывы
└── Источник привлечения

💎 Статус лояльности
├── Уровень (Бронза → Серебро → Золото → Платина)
├── Баланс баллов
├── История начислений/списаний
└── Доступные бонусы
```

### 2. Управление заказами

#### Жизненный цикл заказа
```
1. Создание заказа
   ├── Выбор клиента/автомобиля
   ├── Описание проблемы
   ├── Приоритет (срочный/плановый)
   └── Желаемая дата

2. Диагностика
   ├── Назначение механика
   ├── Первичный осмотр
   ├── Фото/видео дефектов
   └── Формирование списка работ

3. Согласование
   ├── Автоматический расчет сметы
   ├── Поиск запчастей (интеграция UMAPI)
   ├── Сравнение цен от разных поставщиков
   ├── Отправка сметы клиенту (Email/SMS/Push)
   └── Подтверждение клиентом

4. Выполнение работ
   ├── Заказ запчастей
   ├── Обновление статуса (в работе, ожидание детали)
   ├── Чек-листы выполнения
   └── Отметки времени начала/окончания

5. Завершение
   ├── Финальная проверка
   ├── Генерация документов (акт, счет)
   ├── Начисление баллов лояльности
   ├── Запрос отзыва
   └── Планирование следующего ТО
```

### 3. Каталог запчастей и интеграции

#### UMAPI Integration
- **Поиск по артикулу/OEM/кросс-номерам**
- **Получение цен от 50+ поставщиков**
- **Проверка наличия на складах**
- **Сравнение стоимости доставки**
- **Автоматическое обновление цен (кэш 6 часов)**

#### Внутренний склад
- Учет собственных запчастей
- Движение товаров (приход/расход)
- Инвентаризация
- Уведомления о низких остатках

### 5. Программа лояльности с геймификацией

#### Механика
```
Уровни клиента
├── 🥉 Бронза (0-5,000 потрачено)
│   └── Кэшбэк 3%, скидка 5%
│
├── 🥈 Серебро (5,001-15,000)
│   └── Кэшбэк 5%, скидка 7%, приоритетная запись
│
├── 🥇 Золото (15,001-50,000)
│   └── Кэшбэк 7%, скидка 10%, персональный менеджер
│
└── 💎 Платина (50,001+)
    └── Кэшбэк 10%, скидка 15%, бесплатная диагностика

Начисление баллов
├── 1 балл = 2 рубля (50% стоимость)
├── Бонус за первый визит: +500 баллов
├── Бонус за рекомендацию друга: +1,000 баллов
├── День рождения: x2 баллы
└── Сезонные акции (зима/лето)

Использование баллов
├── Оплата до 50% стоимости услуг
├── Обмен на бесплатные услуги (мойка, диагностика)
└── Специальные предложения (шиномонтаж, замена масла)

Геймификация
├── 🏆 Достижения (badges)
│   ├── "Первый визит"
│   ├── "Год с нами"
│   ├── "Привел 5 друзей"
│   └── "Потратил 100,000 рублей"
│
├── 📊 Рейтинги
│   └── Топ клиентов месяца (с призами)
│
└── 🎁 Сюрпризы
    └── Случайные бонусы за активность
```

### 6. Интеллектуальные рекомендации (AI/ML)

#### Что анализируется
- История обслуживания автомобиля
- Средний пробег между визитами
- Типичные поломки для модели/года выпуска
- Климатические условия региона

#### Что предсказывается
```
🔮 Предиктивная аналитика
├── Вероятность поломки (например, стартера) — 68% в следующие 3 месяца
├── Рекомендуемый срок замены расходников
├── Оптимальное время визита (избежать очередей)
└── Персонализированные пакеты услуг

Пример уведомления:
"🔔 Анализ показывает: через ~2,000 км потребуется замена 
тормозных колодок. Забронировать время заранее? 
Это сэкономит вам 10% (раннее бронирование)."
```

### 7. Система уведомлений

#### Каналы
- **Email** — сметы, акты, чеки
- **SMS** — срочные напоминания
- **Push-уведомления** (Android) — статусы заказов
- **Telegram** (опционально) — интеграция с ботом

#### Типы уведомлений
```
Для клиентов
├── Смета готова к утверждению
├── Заказ принят в работу
├── Автомобиль готов к выдаче
├── Начислены баллы лояльности

Для персонала
├── Новый заказ поступил
├── Клиент утвердил смету
├── Запчасть поступила на склад
├── Срок выполнения заказа истекает

```

## Преимущества для бизнеса

### 1. Увеличение выручки на 30-40%

**Как это достигается:**
- **Повышение лояльности** → клиенты возвращаются
- **Автоматические напоминания** → клиенты не забывают о ТО
- **Up-sell и Cross-sell** → система предлагает дополнительные услуги
- **Прозрачность** → клиенты доверяют и приводят друзей

### 2. Сокращение операционных расходов на 20%

**Экономия на:**
- Оптимизации закупок (лучшие цены через UMAPI)
- Сокращении времени оператора (автоматизация)
- Снижении ошибок (меньше переделок)
- Уменьшении холостого времени (умное планирование)

### 3. Рост эффективности персонала

- **Операторы** обрабатывают на 50% больше заказов
- **Механики** получают четкие задачи без путаницы
- **Администратор** видит полную картину за 5 минут

### 4. Контроль и прозрачность

- Все действия логируются (**audit log**)
- История изменений любого заказа
- Защита от мошенничества персонала
- Понимание, где теряются деньги

### 5. Конкурентное преимущество

- **Современный имидж** — цифровой автосервис выглядит солиднее
- **Положительные отзывы** — клиенты ценят удобство
- **Программа лояльности** — удержание клиентов
- **AI-рекомендации** — такого нет у конкурентов

---

## Преимущества для клиентов

### 1. Прозрачность и доверие

- **Видят смету ДО начала работ**
- **Получают фото дефектов** с комментариями механика
- **Следят за статусом** в реальном времени
- **Хранят всю историю** в одном месте

### 2. Экономия времени

- **Онлайн-запись** 24/7 без звонков
- **Напоминания** — не нужно держать в голове
- **Цифровые документы** — не нужно хранить бумаги
- **Быстрая обратная связь** — push вместо ожидания звонка

### 3. Экономия денег

- **Программа лояльности** — реальные скидки
- **Сравнение цен** на запчасти
- **Предсказание поломок** — ремонт до серьезной аварии дешевле
- **Сезонные акции** в приложении

### 4. Персонализация

- **Рекомендации** под конкретный автомобиль
- **Любимый механик** — можно выбрать
- **История в одном месте** — даже при смене телефона
- **Уведомления** только о важном (настраиваемые)

### 5. Спокойствие

- **Ничего не забудете** — система напомнит
- **Контроль бюджета** — история расходов
- **Гарантии** — все документы в приложении
- **Техподдержка** — чат с автосервисом

## Безопасность и надежность

### Безопасность данных

#### Шифрование
- **В транспорте:** HTTPS/TLS 1.3
- **В хранилище:**
  - Пароли: BCrypt с солью (12 раундов)
  - Чувствительные данные: AES-256
  - Ключи: HashiCorp Vault (production)

#### Аутентификация
- **Web:** Session-based (Redis) + CSRF protection
- **Mobile:** JWT (Access 30 мин, Refresh 7 дней)
- **2FA:** (опционально) через SMS/Email/Authenticator

#### Авторизация
- **RBAC:** Role-Based Access Control
- **Гранулярные права:** не только роли, но и permissions
  - Пример: ROLE_OPERATOR может видеть финансы, но не удалять клиентов

#### Защита от атак
- **Rate Limiting:** max 100 запросов/минуту на IP
- **SQL Injection:** Parameterized queries (JPA)
- **XSS:** Sanitization входных данных
- **CSRF:** Token-based для Web
- **DDoS:** Cloudflare WAF (production)

#### Audit Trail
- **Логирование всех действий:**
  - Кто, когда, что изменил
  - IP-адрес, User-Agent
  - До/после значений (для критичных операций)

**Пример лога:**
```json
{
  "timestamp": "2024-01-18T19:45:23Z",
  "user": "operator@autoshop.com",
  "action": "UPDATE_ORDER_STATUS",
  "orderId": "12345",
  "changes": {
    "status": {
      "from": "IN_PROGRESS",
      "to": "COMPLETED"
    }
  },
  "ip": "192.168.1.100",
  "userAgent": "Mozilla/5.0..."
}
```

### Надежность и Отказоустойчивость

#### Резервное копирование
- **PostgreSQL:** Автоматический ежедневный backup (pg_dump)
- **MinIO:** Replication или S3 versioning
- **Redis:** RDB snapshots каждые 6 часов
- **Retention:** 30 дней локально, 1 год в облаке

#### High Availability (для Enterprise)
```
Production Setup:
├── Load Balancer (NGINX)
│   └── 2+ Backend instances (Spring Boot)
│
├── Database (PostgreSQL)
│   ├── Primary (Write)
│   └── 2x Replicas (Read)
│   └── Failover: автоматический (Patroni/Stolon)
│
├── Cache (Redis)
│   ├── Master
│   └── Replica
│   └── Sentinel для failover
│
└── Object Storage (MinIO)
    └── Distributed mode (4+ nodes)
```

#### Мониторинг
- **Healthchecks:** `/actuator/health` каждые 30 секунд
- **Alerts:**
  - CPU/Memory \u003e 80% → Slack/Email
  - Database latency \u003e 500ms → SMS
  - Disk space \u003c 10% → критическое
  - Error rate \u003e 5% → критическое

#### Disaster Recovery
- **RTO (Recovery Time Objective):** \u003c 4 часа
- **RPO (Recovery Point Objective):** \u003c 1 час потери данных
- **Playbook:** пошаговая инструкция восстановления

## Масштабируемость

### Вертикальная (Scale Up)
- **Backend:** увеличение CPU/RAM сервера
- **Database:** более мощная машина
- **Подходит для:** 1-10 автосервисов (до 10,000 клиентов)

### Горизонтальная (Scale Out)
- **Backend:** несколько инстансов за Load Balancer
- **Database:** Sharding (разделение по регионам/клиентам)
- **Cache:** Redis Cluster
- **Подходит для:** сети автосервисов (100,000+ клиентов)

### Микросервисная архитектура (future)

**Когда потребуется:**
- \u003e 20 автосервисов в сети
- \u003e 100,000 активных клиентов
- Разные команды разрабатывают разные модули

**Разделение:**
```
┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐
│  Client Service │   │  Order Service  │   │ Loyalty Service │
│  (Port 8081)    │   │  (Port 8082)    │   │  (Port 8083)    │
└─────────────────┘   └─────────────────┘   └─────────────────┘
         ↓                      ↓                      ↓
┌────────────────────────────────────────────────────────────┐
│              API Gateway (Spring Cloud Gateway)            │
└────────────────────────────────────────────────────────────┘
         ↓                      ↓                      ↓
   PostgreSQL DB1         PostgreSQL DB2         PostgreSQL DB3
```

**Преимущества:**
- Независимое развертывание
- Изолированные БД (отказ одного сервиса не влияет на другие)
- Разные технологии (например, Loyalty на Python для ML)

### Кэширование на всех уровнях

```
┌──────────────────────────────────────┐
│  Level 1: Browser Cache              │
│  (Static assets: JS, CSS, images)    │
└──────────────────────────────────────┘
                ↓
┌──────────────────────────────────────┐
│  Level 2: CDN (Cloudflare)           │
│  (HTML, public API responses)        │
└──────────────────────────────────────┘
                ↓
┌──────────────────────────────────────┐
│  Level 3: Application Cache (Redis)  │
│  (Parts catalog, user sessions)      │
└──────────────────────────────────────┘
                ↓
┌──────────────────────────────────────┐
│  Level 4: Database Query Cache       │
│  (PostgreSQL internal)               │
└──────────────────────────────────────┘
```

### Database Optimization

**Индексация:**
- Все foreign keys
- Поля для фильтрации (status, date)
- Полнотекстовый поиск (GIN index)

**Partitioning:**
- Таблица `orders` разделена по датам (каждый месяц = partition)
- Архивирование старых данных (\u003e 2 года → archive DB)

**Connection Pooling:**
- HikariCP (max 20 connections)
- Реиспользование соединений
