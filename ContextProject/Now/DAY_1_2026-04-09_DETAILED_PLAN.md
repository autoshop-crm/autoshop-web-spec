## Что изучить по технологиям, чтобы понимать общий принцип

Ниже не нужно изучать все идеально. Цель: понять, как каждая технология устроена на уровне "что это, зачем это, как она связана с проектом".

### 1. Java 17

Что изучить:
- классы, интерфейсы, наследование
- `record`, `enum`, `Optional`
- коллекции: `List`, `Set`, `Map`
- исключения: checked / unchecked
- `Stream API` на базовом уровне
- `LocalDate`, `Instant`, `Timestamp`

Что важно понять:
- DTO, entity, service и controller это обычные Java-классы с разной ролью.
- Большая часть Spring-кода строится на хорошей базе Java.

### 2. Spring Boot

Что изучить:
- что такое `starter`
- как работает автоконфигурация
- `@SpringBootApplication`
- `application.properties`
- профили: `application-local.properties`, `application-dev.properties`
- lifecycle запуска приложения

Что важно понять:
- Spring Boot убирает много ручной настройки и собирает приложение вокруг конфигурации и бинов.

### 3. Spring Core / Dependency Injection

Что изучить:
- `@Component`, `@Service`, `@Repository`, `@Controller`, `@RestController`
- что такое bean
- constructor injection
- зачем интерфейс + реализация

Что важно понять:
- объектами управляет контейнер Spring, а не вы создаете их руками через `new`.

### 4. Spring Web

Что изучить:
- `@RequestMapping`, `@GetMapping`, `@PostMapping`, `@PutMapping`, `@DeleteMapping`
- `@RequestBody`, `@PathVariable`, `@RequestParam`
- `ResponseEntity`
- HTTP-коды: `200`, `201`, `204`, `400`, `404`, `409`

Что важно понять:
- Controller принимает HTTP-запрос, но не должен содержать бизнес-логику.

### 5. Validation

Что изучить:
- `@Valid`
- `@NotBlank`, `@NotNull`, `@Email`, `@Pattern`, `@Size`
- `MethodArgumentNotValidException`
- глобальная обработка ошибок через `@ControllerAdvice`

Что важно понять:
- Валидация нужна на границе API, чтобы плохие данные не попадали внутрь логики.

### 6. Spring Data JPA / Hibernate

Что изучить:
- `@Entity`, `@Table`, `@Id`, `@GeneratedValue`, `@Column`
- связи: `@OneToMany`, `@ManyToOne`, `@OneToOne`
- `fetch = LAZY`
- разница между entity и DTO
- repository methods по имени

Что важно понять:
- JPA позволяет работать с БД через объекты, но схема БД все равно остается важнее.

### 7. PostgreSQL

Что изучить:
- таблицы, типы данных, primary key, foreign key
- `unique`, `not null`, `default`
- индексы
- простые `select`, `insert`, `update`, `delete`
- как проверить таблицы после миграции

Что важно понять:
- БД не просто "хранит данные", она защищает целостность системы через ограничения.

### 8. Liquibase

Что изучить:
- что такое migration
- `changeset`
- порядок применения миграций
- rollback на базовом уровне
- почему нельзя править схему руками мимо миграций

Что важно понять:
- Liquibase хранит историю изменений схемы, чтобы проект можно было воспроизвести на любой машине.

### 9. Docker и Docker Compose

Что изучить:
- image
- container
- volume
- port mapping
- environment variables
- `docker compose up`, `docker compose down`, `docker ps`, `docker logs`

Что важно понять:
- Docker Compose нужен, чтобы быстро поднимать одинаковое окружение без ручной установки всего стека.

### 10. Redis

Что изучить:
- что такое in-memory storage
- key-value модель
- где Redis полезен в backend
- TTL / expiration

Что важно понять:
- В этом проекте Redis не главная БД, а вспомогательный быстрый слой для cache и security-сценариев.

### 11. Kafka

Что изучить:
- producer
- consumer
- topic
- event
- почему сообщения идут асинхронно

Что важно понять:
- Kafka нужна, чтобы сервисы обменивались событиями без жесткой связки друг с другом.

### 12. MinIO

Что изучить:
- object storage
- bucket
- upload / download
- presigned URL

Что важно понять:
- MinIO в проекте играет роль локального S3-совместимого файлового хранилища.

### 13. SMTP / Mailhog

Что изучить:
- что такое SMTP
- как приложение отправляет email
- зачем нужен Mailhog локально

Что важно понять:
- Mailhog нужен не для продакшена, а чтобы безопасно видеть письма локально во время разработки.

## Что не нужно учить слишком глубоко в первый день

- тонкости Spring Security
- продвинутое кеширование Redis
- сложные Kafka consumer group стратегии
- MinIO policy management
- оптимизация SQL
- Testcontainers

Это все понадобится позже, но не должно тормозить 9 апреля 2026.
