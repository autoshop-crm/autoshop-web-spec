# Полный план до 1 мая 2026 (без заглушек)

Период: **09.04.2026 -> 01.05.2026**  
Нагрузка: **45-55 часов в неделю** (цель: ~**170-190 часов** суммарно).

## 1) Цель к дедлайну

К **1 мая 2026** должно работать end-to-end:
- `Core` (Client, Vehicle, Order, Parts, Loyalty) с PostgreSQL + Liquibase.
- `AuthService` (JWT access/refresh, роли, проверка токенов).
- `NotificationService` (Kafka consumer + email отправка).
- `FileStorageService` (MinIO upload/download + metadata).
- Интеграции между сервисами (REST + Kafka) в Docker Compose.
- Web и Android в MVP-объеме, но без моков и заглушек.

## 2) Жесткий scope freeze (чтобы уложиться)

Делаем обязательно:
- Полный рабочий backend контур + интеграции.
- Минимально рабочий Web (логин, клиенты, авто, заказы).
- Минимально рабочий Android (логин, мой гараж, статусы заказов).

Не делаем до защиты (если не останется времени):
- AI-рекомендации, расширенная аналитика, сложная геймификация.
- Продвинутые отчеты и нестандартные UX-улучшения.

## 5) Definition of Done к 1 мая

Система считается готовой, если выполнено все:
- `docker-compose up` поднимает все сервисы без ручных правок.
- Реальный end-to-end сценарий проходит:
  - логин,
  - создание клиента,
  - добавление авто,
  - создание заказа,
  - смена статуса,
  - отправка уведомления,
  - загрузка/получение файла,
  - начисление лояльности.
- Есть базовые тесты критичных путей и они проходят.
- Есть краткая документация запуска и демо-сценарий.

## 6) Режим работы, чтобы не сорваться

- Каждый день: 8-10 часов, 1 день в неделю оставлять как полудень/резерв.
- Правило "сначала рабочий поток, потом красота кода".
- Любая задача > 1 дня дробится на 2-3 подзадачи с проверкой результата в тот же день.
- С 30.04.2026 запрещено добавлять новый функционал, только стабилизация.
